## 关键概念

### 三次握手 (30秒理解)
```
客户端                          服务器
   |                               |
   |────── SYN ──────────────────>|  1. "我想连接"
   |                               |
   |<───── SYN+ACK ───────────────|  2. "好的，我也想连接"  
   |                               |
   |────── ACK ──────────────────>|  3. "好的，连接建立"
   |                               |
   └── 连接建立 (ESTABLISHED) ────┘
```

### 四次挥手 (30秒理解)
```
客户端                          服务器
   |                               |
   |────── FIN ──────────────────>|  1. "我要断开了"
   |                               |
   |<───── ACK ───────────────────|  2. "收到，请稍等"
   |                               |  (服务器可能还有数据)
   |<───── FIN ───────────────────|  3. "我也要断开了"
   |                               |
   |────── ACK ──────────────────>|  4. "好的，再见"
   |                               |
   └── 连接关闭 (CLOSED) ─────────┘
```

## 常见问题

**Q: 为什么要三次握手？**  
A: 为了确保双方都能收发数据，并防止旧连接请求干扰。

**Q: 为什么要四次挥手？**  
A: 因为 TCP 是全双工的，需要分别关闭两个方向的数据流。

**Q: 每个演示运行多长时间？**  
A: 大约 5-10 秒，交互式演示时间由你控制。

## 故障排查

```bash

node --version  # 需要 >= 14.0.0

# 检查端口是否被占用
lsof -i:9999   # Linux/Mac
netstat -ano | findstr :9999  # Windows

# 停止占用端口的进程
kill -9 <PID>  # Linux/Mac
taskkill /PID <PID> /F  # Windows
```
