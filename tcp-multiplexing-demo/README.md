# TCP Multiplexing & Demultiplexing 演示

这个项目演示了 TCP 协议中的 **Multiplexing (多路复用)** 和 **Demultiplexing (多路解复用)** 概念。

## 概念解释

### Multiplexing (多路复用)
- **定义**: 多个应用进程的数据通过同一个网络接口同时发送
- **实现**: TCP 使用不同的**端口号**来区分不同的应用进程
- **位置**: 发送端

### Demultiplexing (多路解复用)
- **定义**: 接收端将收到的数据分发到对应的应用进程
- **实现**: 根据 TCP 头部中的**端口号**将数据分发到正确的套接字
- **位置**: 接收端

### 工作原理

```
发送端 (Multiplexing):
┌─────────────────────────────────────────────┐
│  应用1 (端口 3000) ──┐                      │
│  应用2 (端口 3001) ──┼──→ TCP 复用器 ──→ 网络│
│  应用3 (端口 3002) ──┘                      │
└─────────────────────────────────────────────┘

接收端 (Demultiplexing):
┌─────────────────────────────────────────────┐
│  网络 ──→ TCP 解复器 ──→ 应用1 (端口 3000)   │
│                   ──→ 应用2 (端口 3001)      │
│                   ──→ 应用3 (端口 3002)      │
└─────────────────────────────────────────────┘
```

## 项目文件

### server.js
- 启动多个 TCP 服务器，监听不同端口
- 展示 Demultiplexing: 根据端口号分发数据到不同服务

### clients.js
- 多个客户端同时连接到不同的服务器
- 展示 Multiplexing: 多个应用通过同一网络接口发送数据

### detailed-demo.js
- 完整演示 Multiplexing 和 Demultiplexing 的工作流程
- 包含详细的控制台输出和可视化说明

## 使用方法

### 1. 基本演示 (推荐)
```bash
node detailed-demo.js
```

这个脚本会自动启动服务器和客户端，展示完整的多路复用/解复用过程。

### 2. 手动演示

**终端 1** - 启动服务器:
```bash
node server.js
```

**终端 2** - 启动客户端:
```bash
node clients.js
```

### 3. 分别测试各个组件

```bash
# 只启动服务器
npm start

# 只运行详细演示
npm run demo

# 只运行客户端
npm test
```

## 实验观察

运行时注意观察：

1. **Multiplexing**: 多个客户端同时发送数据
2. **Demultiplexing**: 服务器根据端口号将数据分发到正确的服务
3. **并发性**: 不同端口的数据不会互相干扰
4. **独立性**: 每个连接都是独立的，有自己的状态

## 技术要点

### TCP 套接字对
每个 TCP 连接由四元组唯一标识：
- 源 IP 地址
- 源端口号
- 目的 IP 地址  
- 目的端口号

### 端口范围
- **Well-known ports**: 0-1023 (需要管理员权限)
- **Registered ports**: 1024-49151
- **Dynamic ports**: 49152-65535

### Node.js 相关 API
- `net.createServer()`: 创建 TCP 服务器
- `net.createConnection()`: 创建 TCP 客户端
- `socket.on('data')`: 接收数据
- `socket.write()`: 发送数据
